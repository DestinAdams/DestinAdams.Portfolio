Activity Tracker

Develop a simplified version of an activity tracking device (e.g., Fitbit) on an FPGA. The project involves:

  • Embedded input generation for testing hardware designs.
  • Displaying meaningful output on seven-segment displays.

The circuit consists of the following parts:

  • Buttons: Used to start or stop a pulse generator.
  • Pulse Generator: Generates pulses representing steps, depending on mode settings.
  • Fitbit Tracker Module: Counts the pulses (steps) and performs measurements based on the count.
  • Display Controller: Converts binary step count to BCD and displays the count on a 7-segment display.
  • Inputs from buttons must be debounced.
  • All user-created modules must include a synchronous reset.
  • All synchronous modules must use the 100 MHz system clock.

Pulse Generator (Milestone 1)

Inputs: MODE, START, STOP, system clock, reset.

Generates pulses based on mode:

  • Walk mode (MODE = 2’b00): 32 pulses/sec.
  • Jog mode (MODE = 2’b01): 64 pulses/sec.
  • Run mode (MODE = 2’b10): 128 pulses/sec.
  • Off mode (MODE = 2’b11): No pulses.

Inputs: Pulse sequence, system clock, reset.

Functions:

  • Total Step Count: Counts pulses up to 9999, then asserts OFLOW. Steps continue counting for distance calculation.
  • Distance Covered: Displayed in 0.5-mile increments up to 9.5 miles. Distance calculation based on 2048 steps per 0.5 mile.

Converts step count from binary to BCD using the "double dabble" algorithm. Alternatively, the tracker can count directly in BCD, eliminating the need for conversion.

Displays information on the 7-segment displays. Switches between digits at approximately 1 kHz.

Display Specifications

Information displayed in a rotating sequence every 2 seconds: Total step count, Total distance, Mode. Saturate display at 9999 for step count but continue accurate distance calculation.

Utilize IP cores for debouncing START and STOP buttons. Connect IP cores using Xilinx Vivado's Block Design.
